一

存储技术（Storage Technologies）

1.1 RAM 的两种类型

RAM（随机访问存储器）的基本存储单元是一位的 cell，多片 RAM 芯片组成内存。

类型晶体管/位访问速度需要刷新ECC成本用途
SRAM（静态） 4 或 6 1× （基准） ❌ 不需要 可选 ~100× Cache 缓存
DRAM（动态） 1 10× ✅ 需要 需要 1× 主存、帧缓冲
💡
SRAM 访问速度快、无需刷新，但面积大、成本高，适合做 Cache。DRAM 密度大、成本低，但需要定期刷新，适合做主存。

1.2 非易失性存储（Nonvolatile Memory）

DRAM 和 SRAM 都是易失性（Volatile）存储器，断电后数据丢失。非易失性存储器断电后可保留数据：

类型特点
ROM生产时编程，只读
PROM可编程一次（Programmable ROM）
EPROM可用紫外线/X射线批量擦除
EEPROM可电擦除
Flash MemoryEEPROM 的块级擦除版本，约 10万次写后损耗

应用：固件（BIOS、设备控制器）、SSD、磁盘缓存等。

1.3 增强型 DRAM

  • SDRAM（同步DRAM）：使用时钟信号代替异步控制，可复用行地址（RAS, CAS, CAS, CAS…）
  • DDR SDRAM：双边沿时钟，每个时钟周期每个引脚传输 2 位
    • DDR：预取缓冲 2 位
    • DDR2：预取缓冲 4 位
    • DDR3：预取缓冲 8 位（Intel Core i7 标准）

总线结构与内存读写事务

2.1 总线结构

总线（Bus）是一组传输地址、数据和控制信号的并行导线，通常由多个设备共享。

2.2 内存读事务（Load 操作）

以 movq A, %rax 为例，分三步完成：

  1. Step 1：CPU 将地址 A 放到内存总线上
  2. Step 2：主存从总线读取地址 A，取出 word x，放回总线
  3. Step 3：CPU 从总线读取 word x，写入寄存器 %rax

2.3 读取磁盘扇区（DMA）

  1. CPU 向磁盘控制器端口写入：命令 + 逻辑块号 + 目标内存地址
  2. 磁盘控制器读取扇区，通过 DMA（直接内存访问） 将数据传输到主存，CPU 此时可做其他事
  3. DMA 完成后，磁盘控制器向 CPU 发送中断（Interrupt），通知传输完成
三

机械磁盘（Rotating Disk）

3.1 磁盘结构

  • 盘片（Platter）：磁性涂层的圆盘，两面均可存储
  • 磁道（Track）：盘面上的同心圆
  • 扇区（Sector）：磁道被划分成扇形区域，是读写的最小单位（通常 512 B）
  • 柱面（Cylinder）：所有盘片上相同磁道号的集合
  • 记录区（Recording Zone）：外圈磁道更长，扇区数更多；计算容量时用平均扇区数

3.2 磁盘容量计算

容量 = (字节/扇区) × (平均扇区/磁道) × (磁道/面) × (面/盘片) × (盘片/磁盘) 示例：512 B/sector × 300 sector/track × 20,000 tracks × 2 surfaces × 5 platters = 30,720,000,000 B ≈ 30.72 GB

3.3 磁盘访问时间

T_access = T_seek + T_rotation + T_transfer
3~9 ms
T_seek（寻道时间）
磁头移动到目标磁道
~4 ms
T_rotation（旋转延迟）
等待目标扇区转到磁头下方
= 1/2 × (1/RPM) × 60s

⚠️
关键结论：访问时间主要由寻道时间和旋转延迟决定，传输时间几乎可忽略。
扇区第一个位代价最贵，其余位几乎免费。
磁盘比 SRAM 慢约 40,000 倍，比 DRAM 慢约 2,500 倍。

3.4 逻辑磁盘块

现代磁盘将复杂的物理几何结构抽象为：连续的逻辑块序列 (0, 1, 2, ...)

由磁盘控制器（Disk Controller）维护逻辑块到物理扇区（surface, track, sector）的映射，并负责坏道管理等。这也是"格式化容量"与"最大容量"不同的原因。

四

固态硬盘（SSD）

4.1 SSD 结构

  • 由 Flash 闪存 + Flash Translation Layer（FTL） 组成
  • Flash 组织为：Block（32~128 Pages）→ Page（512B ~ 4KB）
  • 数据以 Page 为单位读写，但擦除以 Block 为单位
  • 写入 Page 前必须先擦除整个 Block，一个 Block 约磨损 10 万次后报废

4.2 SSD 性能（Intel SSD 730）

指标顺序随机
读吞吐量550 MB/s365 MB/s
写吞吐量470 MB/s303 MB/s
平均延迟50 µs (读) / 60 µs (写)—

顺序访问明显快于随机访问（内存层次结构的普遍规律）；随机写较慢（需先擦除整块 ~1ms）。

4.3 SSD vs 机械硬盘

✅ SSD 优势
  • 无机械部件 → 更快、更省电、更耐摔
  • 随机读性能远超 HDD
⚠️ SSD 劣势
  • 有写入寿命（通过 Wear Leveling 缓解）
  • 2015年成本约为 HDD 的 30 倍/字节
五

DRAM 内部读取原理

5.1 DRAM 组织方式

d × w DRAM：共 dw 位，组织为 d 个 supercell，每个 supercell 为 w 位。

例：16 × 8 DRAM = 4行 × 4列，每格 8 位（1字节）。

5.2 读取 Supercell(2,1) 的过程

  1. RAS（行地址选通）：发送行号 2 → 将第2行复制到 row buffer
  2. CAS（列地址选通）：发送列号 1 → 从 row buffer 取出 supercell(2,1) → 发送给 CPU

5.3 内存模块（Memory Module）

64 MB 内存模块由 8 片 8M×8 DRAM 组成，每片提供 8 位，合并输出 64 位宽的数据字。

七

局部性原理（Locality）

局部性是程序天然具备的性质，也是内存层次结构能够工作的根本原因。

时间局部性（Temporal Locality）
  • 最近被访问过的数据/指令，近期很可能再次被访问
  • 例：循环变量 sum 每次迭代都被读写
空间局部性（Spatial Locality）
  • 最近访问的地址附近的数据，近期也可能被访问
  • 例：顺序遍历数组 a[i]，每次地址 +4 字节

7.1 局部性分析示例

C// 基础示例
int sum = 0;
for (int i = 0; i < n; i++)
    sum += a[i];   // a[i]：步长-1 访问 → ✅ 空间局部性
                   // sum：每次迭代访问 → ✅ 时间局部性
return sum;

7.2 二维数组的访问顺序

C（行优先访问 — 好）// C 语言中二维数组按行存储（row-major order）
// 按行遍历：a[0][0], a[0][1], ..., a[0][N-1], a[1][0], ...
// 每次访问相邻内存地址 → ✅ 优秀的空间局部性
for (i = 0; i < M; i++)
    for (j = 0; j < N; j++)
        sum += a[i][j];  // ✅ stride-1，好
C（列优先访问 — 差）// 按列遍历：a[0][0], a[1][0], a[2][0], ...（每次跳 N 个元素）
// 步长为 N，每次访问跨越整行 → ❌ 空间局部性差
for (j = 0; j < N; j++)
    for (i = 0; i < M; i++)
        sum += a[i][j];  // ❌ stride-N，差
🎯
程序员技能：能够看着代码就判断局部性好坏，是专业程序员的核心能力之一。
八

内存层次结构（Memory Hierarchy）

8.1 三大基本性质

  1. 快速存储器：成本高、容量小、功耗大
  2. CPU 与内存的速度差距持续扩大
  3. 优秀程序天然具有良好的局部性

这三条性质完美互补，共同催生了内存层次结构。

8.2 典型内存层次

L0
寄存器（Registers）~0 cycles · <1 KB
↕ CPU 寄存器保存从 L1 Cache 取来的数据
L1
L1 Cache (SRAM)~4 cycles · ~32 KB
↕ L1 Cache 保存从 L2 取来的缓存行
L2
L2 Cache (SRAM)~10 cycles · ~256 KB
↕ L2 Cache 保存从 L3 取来的缓存行
L3
L3 Cache (SRAM)~30-50 cycles · ~8 MB
↕ L3 Cache 保存从主存取来的缓存行
L4
主存 Main Memory (DRAM)~100 cycles · ~GB级
↕ 主存保存从本地磁盘取来的磁盘块
L5
本地磁盘 Local Disk~10M cycles · ~TB级
↕ 本地磁盘保存从远程服务器取来的文件
L6
远程存储 Remote Storage网络延迟 · 无限容量

8.3 大思想（The Big Idea）

🌟
内存层次结构创造了一个大容量的存储池，其成本接近底层廉价存储器，但向程序提供数据的速度接近顶层快速存储器。

这依赖于局部性：由于程序总是倾向于重复访问相同数据（时间局部性）和访问相邻数据（空间局部性），慢速大容量存储器很少成为瓶颈。
九

通用缓存概念（General Cache Concepts）

缓存（Cache）：一个更小、更快的存储设备，作为更大、更慢的存储设备中部分数据的"暂存区"。

在内存层次的每一层 k，第 k 层的快速设备都充当第 k+1 层的缓存。

9.1 缓存命中（Hit）

程序需要第 k+1 层中 block b 的数据，而 block b 已经在第 k 层缓存中，直接返回。

9.2 缓存缺失（Miss）

程序需要 block b，但它不在缓存中。触发以下步骤：

  1. 从第 k+1 层取出 block b（费时！）
  2. 将 block b 存入缓存（放置策略 Placement Policy 决定放哪里）
  3. 如果缓存已满，需要驱逐某个 block（替换策略 Replacement Policy 决定驱逐谁）

9.3 三类缓存缺失

类型原因说明
冷缺失（Cold Miss）
也叫强制缺失		 缓存为空 第一次访问，缓存中什么都没有，不可避免
冲突缺失（Conflict Miss） 多个 block 映射到缓存同一位置 缓存容量够大，但映射规则导致冲突
例：block i 必须放在 i mod 4 号槽位，若反复访问 block 0 和 8，每次都会 miss
容量缺失（Capacity Miss） 工作集（Working Set）超过缓存容量 程序活跃使用的数据量大于缓存大小

本讲小结

  • CPU、内存、外存之间的速度鸿沟持续扩大
  • 优秀程序天然具备局部性（时间局部性 + 空间局部性）
  • 内存层次结构利用局部性，通过缓存弥补速度差距
  • 缓存缺失分三类：冷缺失、冲突缺失、容量缺失
  • 编写缓存友好的代码（如按行访问二维数组）是性能优化的关键
