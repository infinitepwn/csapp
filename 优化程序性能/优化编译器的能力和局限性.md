## gcc的优化命令
-Og 基本优化
-O1,O2,O3会得到更高的优化

但实际上编译器并没有那么强大，因为过度的优化可能会导致程序错误执行，为了符合程序的原义，编译器只使用安全的优化

## 内存别名使用(memory aliasing)
例如下面这个例子
```C
void twdiddle1(long *xp,long *yp)
{
	*xp += *yp;
	*xp += *yp;
}
void twddile2(long *xp,long *yp)
{
	*xp += 2* *yp;
}
```
看上去的效果是一样的，而且，第二个函数的效率看上更高，因为它只需要进行3次内存饮用，读xp，读yp，写xp，而第一个函数需要6次，但是我们要考虑到，xp和yp有可能指向内存的同一片空间

当xp=yp时，第一个函数会使得xp变为原来的4倍，而第二个会变为原来的3倍
