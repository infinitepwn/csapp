读取元素我们可以使用数组而不是反复调用函数
``` C
void combine3(vec_ptr v, data_t *dest)
{
    long i;
    long length = vec_length(v);
    data_t *data = get_vec_start(v);
    *dest = IDENT;
    for (i = 0; i < length; i++){
        *dest = *dest OP data[i]; 
    }
}
```

## combine3 的优化点

`combine2` 仍然在每轮循环里调用 `get_vec_element(v, i, &val)`。这个函数会做边界检查，再把元素写到临时变量 `val`：

```C
int get_vec_element(vec_ptr v, long index, data_t *dest)
{
    if (index < 0 || index >= v->len)
        return 0;
    *dest = v->data[index];
    return 1;
}
```

对于 `combine` 来说，循环变量 `i` 本来就在 `[0, length)` 范围内，所以这次边界检查和函数调用都是额外开销。`combine3` 先用 `get_vec_start(v)` 取得底层数组指针，之后直接访问 `data[i]`，循环体更短：

```text
combine2: 每个元素 = 函数调用 + 边界检查 + 读数组 + 加法
combine3: 每个元素 = 读数组 + 加法
```

在 `/opt/measure_combine_cpe.py` 的实测中，`n=1048576` 时：

```text
combine2: 0.370 ns/element
combine3: 0.068 ns/element
```

当前环境不能读取稳定 CPU 频率，所以图中默认使用 `ns/element`。如果设置 `CPU_HZ=频率`，脚本会自动换算成 CPE。
