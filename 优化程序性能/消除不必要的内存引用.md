## 基本思想

循环中如果反复从内存读写同一个中间结果，就会增加加载和存储指令。优化时可以先把中间结果保存在局部变量里，循环结束后再写回内存。

在 `combine3` 中仍然有一个问题：

```C
for (i = 0; i < length; i++) {
    *dest = *dest OP data[i];
}
```

每轮循环都要读 `*dest`，再把新结果写回 `*dest`。如果改成局部变量：

```C
data_t acc = IDENT;
for (i = 0; i < length; i++) {
    acc = acc OP data[i];
}
*dest = acc;
```

循环体就不再依赖对 `dest` 指向内存的反复读写。这样通常更容易让编译器把 `acc` 保存在寄存器里。

编译器不能总是自动做这个优化，因为 `dest` 可能和 `data` 指向同一片内存，也就是内存别名问题。如果 `dest == &data[i]`，把每轮写回推迟到循环结束会改变程序语义。

这个点和前面的三个版本关系如下：

```text
combine1: 循环内重复求长度 + 每轮调用 get_vec_element + 每轮读写 dest
combine2: 长度移出循环 + 每轮调用 get_vec_element + 每轮读写 dest
combine3: 直接访问数组 + 每轮读写 dest
进一步优化: 直接访问数组 + 用局部变量累积，最后写回 dest
```
